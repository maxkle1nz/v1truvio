# V1TRUVIO — Vitruvian Machine Mind

**Mente geradora de direção de arte diegética para coabitação humano-máquina.**

Não é um design system. É o compilador que gera design systems: uma ontologia de design (tese, inimigo, operador, leis) + uma API estética (6 vetores calibráveis) + um banco de leis de composição (RAG cards) + um pipeline com gates objetivos. Estilo é output; a mente é a gramática.

> Criar interfaces que devolvem ao humano a sensação de operar uma máquina viva, legível e responsável — não para complicar o uso, mas para restaurar presença, tato, consequência e consciência na decisão digital.

## Estado

**Fase 0 — mente compilada, ainda não provada.** Os artefatos existem; a skill de deploy e as provas de vida (P1–P4 do PRD) ainda não. A bancada de testes empírica é o repo [~/almus](../almus/) — onde TESSERA, APOLLO, KRONOS e VERITAS nasceram como experimentos pré-formalização.

## Uso hoje (manual, pré-skill)

Carregue `mind/MANIFESTO.md` + `mind/CONTRACT.md` + `mind/VECTORS.md` + `mind/MODES.md` no contexto do agente de design, mais os 3–5 cards de `mind/rag/` que o diagnóstico pedir. Siga o pipeline do contrato. Toda geração termina com perfil declarado, brief rastreável e prova mínima.
